# 全栈学习路线（结合本仓库）

目标：以“能独立做一个可上线的小产品”为导向，把设计能力转成可落地的前后端实现能力。

本仓库结构：

- 前端：React + Vite + TypeScript + Tailwind（`src/`）
- 后端：Node.js + Express（`api/`）
- 运行方式：`npm run dev` 并行启动前后端；前端通过 Vite 代理访问后端 `/api/*`

## 你需要建立的 3 个心智模型

1. 组件 = 可复用的“设计模块”
   - 把页面拆成稳定的 UI 单元（按钮、卡片、输入框、布局容器）。
   - 每个组件只负责：结构（JSX）+ 状态（state）+ 样式（Tailwind/Token）。
2. API = 前后端之间的“契约”
   - 前端只通过 HTTP 调后端；后端输出稳定的 JSON 结构。
   - 让类型（TypeScript）成为契约的一部分，减少沟通成本。
3. 数据流 = 你在 Figma 里做交互流程的代码版
   - 用户操作 → 触发事件 → 更新状态 → 重新渲染 UI。
   - 复杂后用状态管理（本仓库有 zustand）。

## 从哪里开始读代码（最短路径）

前端：

- `src/main.tsx`：React 启动入口
- `src/App.tsx`：根据 `VITE_APP_MODE` 选择 demo/product
- `src/apps/demo/DemoApp.tsx`：路由（react-router-dom）
- `src/apps/demo/pages/Home.tsx`：请求后端 `/api/health` 的最小例子
- `src/lib/apiClient.ts`：前端请求封装与错误处理

后端：

- `api/server.ts`：本地启动入口（监听端口）
- `api/app.ts`：Express 中间件、路由、错误处理
- `api/lib/http.ts`：统一成功/失败返回结构（供前端解析）

## 4 个循序渐进的练习（每个 30–90 分钟）

练习 1：做一个“你自己的 API”并在前端展示

- 后端：新增 `GET /api/version` 返回 `{ version, sha?, builtAt? }` 之类信息
- 前端：在 Home 页面加一个区域展示该接口
- 学到：路由、JSON 结构、前后端联调、错误状态 UI

练习 2：加一个带校验的表单

- 前端：输入框 + 按钮 + loading/error/success 状态
- 后端：`POST /api/echo` 校验字段（比如 title 必填、长度限制）
- 学到：表单事件、请求体、后端校验、错误码设计

练习 3：把页面拆成 3–5 个组件，并抽一个“设计 token”层

- 提取：`<Card/>`、`<Badge/>`、`<Button/>`、`<Field/>` 等
- Token：比如颜色/圆角/间距规则集中到一个地方（可从 Tailwind class 组合开始）
- 学到：组件边界、复用、可维护性

练习 4：用 zustand 管理一个跨页面状态

- 做一个最小的“项目设置” store（主题、语言、最后一次请求结果等）
- 学到：状态管理的必要性、数据持久化（可选）

## 你作为设计师的优势，如何转成工程优势

- 你擅长“命名”和“信息结构”：把它用在组件名、路由名、API 名、错误码上
- 你擅长“状态设计”：把 idle/loading/success/error 变成明确的 UI 状态机
- 你擅长“约束与规范”：把视觉规范沉淀成 token、组件 API 和可复用布局
