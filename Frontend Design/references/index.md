### 参考图说明

这是一个 `白底黑线`、结构稀疏、留白充足的前端参考图。它相比此前密集、发光、偏 `dashboard` 的版本，更接近我们想要的研究型原型界面。

其核心特征包括：

- 中央是 `蒲公英 / 原则花盘` 结构
- 花盘外层用大量细线与稀疏节点表现原则分布
- 中央圆心显示当前激活的原则数量
- 向下有一根明确的 `workflow` 主轴，将原则收束到流程与最终产出
- 左侧是 `Task Definition`、`Principle Layers`、`System Maturity`
- 右侧是 `Principle Details`、`Related Tests`、`Legend`
- 底部是 `Workflow & History` 与 `Rule Write-back Log`

### 为什么保留这张图

这张图比较符合当前原型想要的方向：

- 不是炫技式的未来控制台
- 不是高度密集的数据看板
- 而是一个 `diagrammatic interface`
- 有清楚的 `panel` 边界
- 保留中心可视化的轻盈感
- 适合承载 `原则 -> 流程 -> 结果` 的原设计逻辑

### 后续可以继续保留的设计要点

- `白色背景`
- `黑 / 灰` 细线系统
- `轻边框` 的左右面板
- 中央 `蒲公英` 结构保持稀疏，不要过密
- 底部 `workflow` 区域保持清楚的流程节点感
- 面板与中央可视化之间保持充足留白

### 后续继续优化的方向

- 将中心结构进一步和 `原则种子 / 花盘 / 茎干 / 结果` 隐喻对齐
- 强化 `高层 / 中层 / 底层原则` 的视觉区分
- 明确 `测试分支` 与 `任务运行记录` 的关系
- 保持界面克制，不滑向复杂 `SaaS dashboard`
