# 访谈任务单模板

适用位置：`RtD Plan` 第三部分“任务执行与观察”

使用方式：每次访谈准备 `2` 个小任务；可根据受访者背景替换任务场景，但建议保持输入材料类型和输出要求大致一致，以保证可比性。

## 一、主持人统一说明

在接下来的这个部分，我会请你用自己比较熟悉、平时也比较常用的工具，完成两个小的信息设计任务。  
你可以自由选择是否使用 AI，也可以使用你平时习惯的 AI 工具。  
在操作过程中，希望你尽量边做边说，说说你现在在想什么、为什么这样做、哪里觉得顺手或不顺手。我们更关注你的过程，不追求最后一定做得特别完整。

## 二、任务单结构模板

每个任务建议包含以下内容：

### 1. 任务名称

示例：`任务 1：将一段活动信息整理成单页 PPT`

### 2. 任务情境

用一句话交代真实使用场景，帮助受访者代入任务。

示例：
- 假设你需要在组会上快速介绍一个项目，请把下面的信息整理成一页适合展示的 PPT。
- 假设你需要为一次活动做一张社交媒体信息卡片，请把下面的信息转化成适合传播的视觉内容。

### 3. 任务目标

明确告诉受访者要完成什么。

示例：
- 请根据给定材料，产出 `1` 页 PPT 信息页。
- 请根据给定材料，产出 `1` 张社交媒体图文卡片。

### 4. 输入材料

这里放原始素材。建议使用非结构化或半结构化信息，以便观察其信息整理过程。

可选材料类型：
- 一段活动介绍文字
- 一段项目说明文字
- 一组数据摘要
- 一段研究结论摘要

### 5. 输出要求

不要写得太死，但要给出基本边界。

建议包含：
- 输出形式：`单页 PPT / 海报 / 信息图 / 社交媒体图文`
- 输出数量：`1` 页或 `1` 张
- 目标受众：如同学、老师、客户、普通用户
- 目标场景：如汇报、传播、宣发、总结

### 6. 工具说明

建议固定这段说明：

- 你可以使用自己平时熟悉的设计工具和 AI 工具。
- 如果你平时不用 AI，也可以按你自然的方式完成。
- 如果你愿意，也可以在过程中尝试新的 AI 工具或 skill。

### 7. Think Aloud 提示

建议固定这段说明：

- 操作时请尽量把你的想法说出来。
- 包括你怎么理解任务、为什么选这个工具、为什么这样改。
- 如果你觉得某一步卡住了，也请直接说出来。

## 五、任务后即时追问

每个任务完成后，可立即追问以下问题：

- 刚刚这个任务里，你最先考虑的是什么？
- 哪一步你觉得最顺手？为什么？
- 哪一步最卡？为什么？
- 如果不用 AI，你觉得这个任务会有什么不同？
- 如果让你重新做一次，你会换一种方式吗？

## 六、记录建议

建议每个任务同步记录以下信息：

- 开始时间 / 结束时间
- 使用工具
- 是否使用 AI
- 首次输入内容或 prompt
- 关键修改节点
- 受访者原话
- 关键痛点
- 最终产出截图或文件

## 七、观察重点表

| 关注类型 | 具体内容 |
| --- | --- |
| 任务理解 | 他拿到任务后先看什么；他如何判断重点信息；他会不会先自己重组内容，再交给 AI。 |
| 工具选择 | 他为什么用这个工具；是一上来就想到，还是试了几个才决定；不同工具在他心里分别承担什么角色。 |
| 与 AI 的对话方式 | 他怎么描述需求；是先说内容、先说结构，还是先说视觉风格；prompt 是一次说清，还是边试边改；哪类需求最难表达给 AI。 |
| 修改与迭代行为 | 第一次结果出来后，他先改什么；是改 prompt、改素材、换工具，还是自己手动补；他会反复试错几轮；哪一步最容易卡住。 |
| 控制权与分工 | 哪些部分他愿意交给 AI；哪些部分他一定要自己控；他把 AI 当成灵感助手、排版助手，还是生成执行者。 |
| 情绪与判断 | 哪些时刻他明显犹豫、不满、烦躁、失望；哪些时刻他会说“这个还不错”；他评价结果时最看重什么，比如效率、结构、视觉、可控性、可编辑性。 |
| 绕路与补救策略 | 当 AI 效果不好时，他会不会换一种说法、换工具、换素材，或者干脆自己手动完成；这些补救动作出现在哪些节点。 |
| 结果接受度 | 他最后是否接受 AI 结果；接受的是哪一部分，不接受的是哪一部分；最终成品中 AI 产出占了多大比例。 |
