# P8 用户访谈逐字稿（郑炜钰）

主持人（00:00:54）
同学你好，我也是同济大学的，现在是研二交互设计方向，最近在做毕设，访谈是我毕设前期的用户访谈，用于挖掘有关 AI 辅助设计的痛点，还有大家对于我毕设概念的一些想法。本次访谈大概分为三个部分：一是了解你在 AI 辅助设计以及信息设计方向的经验和痛点；二是结合我发给你的 markdown 文档做一个小的信息设计任务，过程中你也可以大声说出你的想法；三是结合我毕设当中的元设计概念来谈谈它的具体应用和评价。本次访谈会录音录屏，你的个人信息只会用于我的毕设内容，相关信息也会匿名处理。

郑炜钰（00:02:34）
OK，我是现在同济大学研一交互设计。AI 工具使用的话，现在用得最多是 Codex，做一些原型设计，比如搭建网站、APP，还有信息处理、信息整合，做一些效果可视化，各种编程代码，还有硬件烧录都会用到。使用频率可以说是每天都在用。

主持人（00:03:08）
了解，那你具体搭建网站，整个流程大概是怎样的？

郑炜钰（00:03:18）
我现在也是在探索中，包括在公司和课程作业当中都在用。我现在搭建一个工具的说明书网站，遇到了一些问题，记忆文件堆得越来越大，token 烧得非常快，上下文丢失的幻觉也越来越多，哪怕每次都让他读取新的记忆文档也还是有问题，没有找到可以兼得的方法——既让它不丢失上下文，又不那么消耗 token。现在 100% 积分 5 个小时只能进行 2~3 个新的对话就用完了。

主持人（00:04:30）
我感觉你可以试试 Claude Code，它有 100 万的上下文，如果你对上下文这件事情有执念的话可以试试那一套模型。那比如说在整个工作流上有什么探索吗？从 0 到 1 搭建这个网站，大概会分为哪些阶段？

郑炜钰（00:05:10）
我发现了更新的一种搭建方式，不要纯粹让 Codex 去做整件事。我先让一个 AI 整理信息，把所有需求和文档丢给它，让它真正理解，然后生成一个目标规划——一个长链，然后它可能会搭建一个小的 agent，也可能搭建一个记忆文档，然后我把它丢给下一个对话去做。我发现谷歌 AI Studio 可能更擅长这块，上下文更多，处理速度也更快。

主持人（00:06:25）
那具体到后续生成和迭代，有遇到过什么问题吗？

郑炜钰（00:06:30）
目前进行的说明书网站和个人网站还处于 0.1 或者 0.5.1 的过程当中，还没有到迭代这一块。如果是维护的话，更多是专门开一个对话去让他把代码优化一下，文档越多他可能规范性就没那么好。

主持人（00:07:07）
比如说在日常课程当中，有尝试过用 AI 给你直接生成 PPT 或者 HTML 去做汇报吗？

郑炜钰（00:07:19）
有，升 PPT、升展板，还有让他连 MCP 那个 Figma 的东西，升那种可编辑的东西都有。

主持人（00:07:34）
这些主要是在什么场景下，和手搓比起来有什么优势？

郑炜钰（00:07:43）
他们的优势就是我可以在旁边玩手机刷抖音，我只需要打开语音，跟他说我要干什么，把东西丢给他，然后一直让他接着去迭代去做就够了。像做 PPT 这块，我觉得 Manus 会更好用一些，但它没有办法配置 MCP，所以如果要做成 Figma 那种可编辑的，局限性会比较大。

主持人（00:08:25）
你当时是怎么探索下才发现 Manus 适合做 PPT？

郑炜钰（00:08:33）
亲自使用，它生成的东西会更加好一些。不管是生成速度还是质量，它都比豆包或者 Codex 要牛一点，可能跟 Code 上的一些工作流会持平。

主持人（00:08:54）
他们生成的东西，大概百分之多少符合你的要求？有什么局限没办法帮你生？

郑炜钰（00:09:11）
比如说生成图片，除非像用 Codex 我必须要安装特定的好多个 skill，它才能达到我想要的水平。用它做 PPT 效果还是很差，上下文丢失很快，烧 token 非常多，生成速度也很慢，生出来的东西还是一坨，只能说能用，但是远不如谷歌 AI Studio 或者 Manus 去做这类东西。

主持人（00:10:24）
视觉信息呈现上，他写的一些字段有没有达到你的要求？

郑炜钰（00:10:32）
除非用另外一种方式，用 image to 直接生成图片的话，效果可能会更好一些。还有谷歌的 Notebook LM，它这个东西生成创造性的图片会更 OK 一点。

主持人（00:11:49）
接下来我们就可以开始做小任务了。背景是小红书和同济联合开一个主题会，探讨 AI 和设计师之间协作的话题，我会提供基础信息内容，你可以生成一张 A4 大小的海报。过程中你可以先对内容的信息层级、结构、风格进行简单定义，在此基础上让它去生成初版海报，如果不满意可以进行 2~3 轮迭代。

郑炜钰（00:12:45）
OK，那我可以拿 GPT 来跑图片可以吧？它已经可以生成 PSD 文档了，可以把生成的图片进行图层拆分。不行的话，我就把它丢给 Codex 重新拆一遍。最近 Codex 已经可以控制你的电脑，打开 PS 去帮你做海报了，然后同时它可以通过内置浏览器去搜索素材，然后丢进 PS 里面。

郑炜钰（00:19:06）
按照我的习惯，我会直接把东西全部丢给他，让他先理解我的目的和背景，不会分段去做。然后我会说，我们先不要讨论任务二，我们先专注任务一，理解我的需求，不要直接给我答案。

郑炜钰（00:20:32）
有可能在进行任务之前，我自己会先给出一个很长的 prompt，包含需求和目标，但现在 AI 能力我还没有摸清楚能力边界，它有可能已经不需要了。一般会进行 1~3 轮来回讨论，才会确认我到底想要什么样的东西。

郑炜钰（00:27:14）
怎么那么慢？好丑！

主持人（00:27:31）
为什么会长这样？你可以看一下他风格是怎么定义的。

郑炜钰（00:27:34）
原则上来说他不应该做这么难看，一般这种时候我就开始骂他，然后重新回到任务一，让他重新做一遍。

郑炜钰（00:29:26）
就像那种 web coding 出来的网站，框里面是这个东西，框框框一层套一层。他自己都能发现这个问题——他自己都说这不是宣传海报，是卡片 UI——为什么还做出一个卡片 UI 出来？

郑炜钰（00:31:29）
方案 B 好像更好一些。按照我再古早一点的工作流，我就是先拿 image 升图片，升完之后再丢给另外一个工具去拆图层，但现在不用了，让它升 PSD 就好。

郑炜钰（00:34:37）
中文大字，让他极致成这种程度就够了。

主持人（00:35:54）
我觉得你这个步骤有点像，抛开视觉层的东西，先去把基础的信息和结构对自己当时的要求定下来。

郑炜钰（00:36:06）
对，因为我现在做的东西基本上会先把视觉层面的东西全部抛弃掉，因为我觉得视觉是最后一步的一个东西了，现在视觉越来越不值钱了。我现在其实用 AI 工具还属于探索期间，更多是为了探索而探索。

郑炜钰（00:37:46）
现在已经完全不需要给他一个身份了，就比如说"你现在扮演一个设计资深专家"，完全不需要了，除非我想要他以特定风格回答。如果是用 Codex 的话就完全不需要了，他只需要理解我的任务目标是什么，甚至直接打开 Go 模式。

主持人（00:53:57）
假如说让你去迭代这一张海报，你会做哪些事情？

郑炜钰（00:54:16）
比如说像这种字体排版排得特别难看，或者字体不好，我就会人工去调，哪怕是 Lovart 做的东西非常牛逼，有些字体还是需要人工做的，也可能要从自己的字库里找。但现在 Lovart 也在继续迭代了，只需要给他一些参考，他可以通过参考自动生成可编辑的字体。

主持人（00:55:07）
你整个过程有遇到什么问题吗？

郑炜钰（00:55:12）
PSD 文件生成有问题，它的一些操作是黑箱，我看不见它到底干了什么，以及没干什么。

主持人（00:55:21）
如果黑箱能够被解决，你最希望是怎样一个理想的状态？

郑炜钰（00:55:27）
他已经解决了，对，在 Codex 里他会直接在右边去做，生成一个他自己的鼠标在旁边点击，让我看到他到底在干什么。我就会在这边开一个他的机器文件可以查看，或者底下再开一个终端，这样我可以知道他到底在做什么，以及他在干一些事情的时候，我也可以直接去打断，说你这一步可以往那边调整。虽然这样会更烧 token，但起码内容是可控的，我们的大方向不会因为他的误解而改变。

主持人（00:56:20）
那反过头来说，如果信息设计你比较熟练，你会前置地去定义一些规则，让 AI 去效仿你的规则和方法来做执行吗？

郑炜钰（00:56:52）
我这一条还没有跑通。我可能在我现在实习的工作当中，当我把这一份内容跑通之后，我会把自己烧成一个 skill，然后喂给我自己电脑，让它去按照我这个工作流接着去做。

主持人（00:57:06）
skill 还是蛮重要的，你说的问题是什么？

郑炜钰（00:57:35）
再过一段时间可能会有新的概念，我不着急去研究这一块东西，因为 skill 其实也才刚出来几个月。

主持人（00:57:46）
skill 它包含了最基础的一些规则、assets、reference、workflow，这些东西上手起来其实蛮方便的，或许这些能力本身也是可以迁移的。

郑炜钰（00:57:59）
本质上它其实就是一堆文字、markdown 文档什么的。但是谁知道未来一个月两个月三个月会不会还有新的东西。

主持人（00:58:30）
假如说有一个 skill 来给你做整个过程，你会期待他做成什么样？他在做信息设计的时候，可能会给你造成怎样的困扰？

郑炜钰（00:58:48）
我觉得肯定还是看能力，虽然他只是一堆文字，但是他的一些工作流可能会完全颠覆性。就像最 low 的一种东西——纯文字排版——但如果有很好的工作流，它可能会自动去找一些很好的素材，或者自己去生图，生完图再放入里面去排版，什么都有可能。

主持人（01:01:13）
你会觉得信息设计值得被抽象出哪些规则，让 AI 生成能够更加约束它的生成？

郑炜钰（01:01:21）
比如说一些可视化排版的边界，以及一些字体使用，还有产品留白什么乱七八糟的东西，它都是需要把一些现有的东西烧给它的。

主持人（01:01:44）
最后一个问题，你对于 AI 辅助设计还有怎样的畅想吗？

郑炜钰（01:01:55）
畅想可能更多是希望在技术这一块，还有一些多模态领域，希望它可以有越来越多的工具。就是我们现在刚刚打通了 AI 工具跟其他软件之间的一些边界，可以通过 MCP，可以直接操控电脑去做这种东西，到未来我觉得他可能会打破更多的闭源障碍，帮控更多的东西。目前还属于这么一个阶段，未来可能会有更多的模态，甚至是在自己的生活里。
